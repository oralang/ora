  v0.2 is primarily a compiler/FV correctness release, with a large debugger track alongside it. The important work is not just ADTs and debugger features: the Z3 verifier was materially hardened with unsat cores, proof
  plumbing, vacuity detection, fail-closed degradation, removal of unsafe opaque/product fallbacks, and much stricter SMT width/type discipline. We also started the traits surface with parser, AST, sema, extern trait
  calls, ABI/SIR lowering, trait ghost blocks, impl checking, generic trait bounds, and associated trait method calls.

  Formal Verification / Z3
  The verifier changed substantially.

  - Added explain-mode unsat cores, including tracked assumption proxies for user assumptions, imported callee obligations, imported callee ensures, environment/frame assumptions, and ghost axioms.
  - Added optional Z3 proof generation plumbing and report support. Raw proof payloads are emitted in JSON when enabled, while markdown omits raw proof bodies.
  - Added optional unsat core minimization.
  - Added vacuity detection for inconsistent assumptions. Contradictory requires now appears as vacuous in reports instead of silently looking like a normal proof.
  - Added inconsistent-base handling: if a function’s base assumptions are inconsistent, later obligations are skipped/reported conservatively instead of producing misleading proofs.
  - Added ghost-axiom consistency checks. Inconsistent ghost axioms now fail verification even when there are no ordinary obligations.
  - Hardened UNKNOWN behavior. Unknown solver results fail closed and are reported as verifier errors rather than treated as success.
  - Hardened mid-proof degradation. If the encoder degrades after query preparation or during proving, verification aborts with EncodingDegraded.
  - Tightened degradation reporting with persistent reason strings and bounded aggregation of degradation causes.
  - Added fail-closed regression coverage for struct metadata loss, try-statement exactness loss, read/write-set loss, product metadata loss, and unsupported coercions.
  - Added sequential/parallel equivalence checks and defensive fail-closed paths for parallel result collection.

  SMT Encoding / Soundness
  The encoder became much stricter.

  - Tightened bitvector width discipline: signed/unsigned coercions, narrowing, sequence length to BV, typed numerals, and mixed-width comparisons now require explicit/sound coercion paths.
  - Added checked power overflow obligations and regressions.
  - Removed or retired unsafe product/tuple opaque fallback behavior. Product values now use exact datatype metadata when available and fail closed when metadata is missing.
  - Added SMT datatype support for products: named structs, anonymous structs, and tuples use per-sort field accessors instead of global field UFs.
  - Added SMT datatype support for tag-only enums.
  - Migrated error unions from opaque/tuple-like encoding toward datatype constructors, including per-error constructors and discriminant recovery.
  - Fixed Result/error-union propagation cases: named payload roundtrip, multi-error try, wide catch propagation, legacy error returns, and enum constants with repr-backed storage.
  - Improved known-callee summary exactness: path assumptions, requires/ensures, read/write slot metadata, state replay, and helper summaries.
  - Remaining opaque UF behavior is now narrower: unresolved/metadata-absent helper cases are explicit fail-closed boundaries, not silent proof paths.

  ADT / Result / Enum Work
  The ADT track progressed through several phases.

  - Added !ora.adt IR support.
  - Added Result constructors and match lowering.
  - Added public ABI lowering for Result and multi-error cases.
  - Added payload-error lowering and named error payload support.
  - Added enum repr-aware behavior, explicit string/bytes enum values, and comptime enum discriminant expressions.
  - Added source ADT constructors through SIR.
  - Added ADT enum pattern matching, sum-match exhaustiveness, or-patterns, discard patterns, wildcard warnings, and aggregate payload matching.
  - Added comptime ADT matching and comptime payload persistence.
  - Added refinement support for ADT payloads and aggregate guards.
  - Added OraToSIR ADT carrier helpers, centralized materialization kinds, ADT handle malloc/load helpers, wide ADT decode fixes, and return/cast lowering refactors.

  Comptime
  The comptime system was expanded beyond simple constant folding.

  - Strengthened phase 1 comptime evaluation.
  - Added diagnostics for comptime limits.
  - Locked in partial evaluation behavior.
  - Completed broader comptime evaluation and partial folding.
  - Added bounded runtime loop unrolling for small constant for and while loops.
  - Added nested runtime unroll provenance and tighter policy.
  - Added comptime Result matching, nested struct field updates, ADT payload values, and trait-associated comptime method execution.

  Traits
  Traits are an important new surface in v0.2, but they should be described honestly as “started and partially implemented,” not fully complete.

  Implemented/tested pieces include:

  - Parsing and AST lowering for trait and impl.
  - extern trait parsing with call / staticcall methods.
  - Trait method signatures, receiver kinds, comptime methods, and trait ghost blocks.
  - Impl conformance checks: missing methods, extra methods, wrong parameter types, wrong return types, duplicate impls.
  - Extern trait calls through external<Trait>(addr, gas: ...).
  - Declared extern trait errors, including payload-bearing errors.
  - ABI signature and selector computation for extern traits.
  - Lowering extern trait calls to ABI/external-call ops and through SIR.
  - Dynamic bytes/string returns, narrow integer returns, static struct returns, tuple returns.
  - Trait ghost facts collected and verified.
  - Trait bounds on generic functions.
  - Resolution of trait-bound methods in generic bodies.
  - Associated trait calls and concrete impl symbol lowering.
  - Comptime associated trait methods and receiver trait methods.

  Not complete / still needs care:

  - Full trait system design is not closed.
  - Runtime polymorphism is not being introduced.
  - This is compile-time/generic/extern-call oriented trait support, not a full dynamic dispatch object model.

  Honest Status
  v0.2 materially improves soundness and compiler coverage. The strongest claims are:

  - SMT degradation is much more consistently fail-closed.
  - Unsat cores/proofs/vacuity reporting are now real verifier features.
  - Product/enum/error-union encodings are no longer relying on the old broad opaque paths.
  - ADT/Result lowering is substantially usable across HIR, MLIR, SIR, SMT, and comptime.
  - Traits have a serious first implementation slice, but they are not “done” as a whole language feature.
  - Debugger functionality is much broader, but DAP integration is still partial.
